# Визуальный Редактор Промптов

Современный веб-интерфейс для работы с библиотекой промптов для вайбкодеров.

## 🚀 Быстрый старт

### Локальный запуск

1. **Простой способ** - откройте файл напрямую:
   ```bash
   # Откройте в браузере
   open editor/index.html
   # или
   firefox editor/index.html
   ```

2. **С локальным сервером** (рекомендуется):
   ```bash
   # Python
   cd editor
   python -m http.server 8000

   # Node.js
   npx serve editor

   # PHP
   php -S localhost:8000 -t editor
   ```

   Затем откройте http://localhost:8000

## ✨ Возможности

### 📖 Режим Просмотра
- Красивый рендеринг Markdown
- Подсветка синтаксиса для code blocks
- Удобная навигация по категориям
- Поиск по промптам

### ✏️ Режим Редактирования
- Полноэкранный редактор с syntax highlighting
- Автосохранение (в разработке)
- Создание новых промптов
- Редактирование существующих

### 🔀 Разделенный Режим
- Редактор + Live Preview
- Мгновенное обновление предпросмотра
- Подсветка кода в реальном времени

### 🛠️ Дополнительные функции
- **Копирование промпта** - одна кнопка для копирования в буфер
- **Экспорт** - сохранение промптов в .md файлы
- **Категории** - организация по типам
- **Поиск** - быстрый поиск по названию и описанию
- **Темная тема** - удобная для глаз

## ⌨️ Горячие клавиши

| Клавиши | Действие |
|---------|----------|
| `Ctrl/Cmd + 1` | Режим просмотра |
| `Ctrl/Cmd + 2` | Режим редактирования |
| `Ctrl/Cmd + 3` | Разделенный режим |
| `Ctrl/Cmd + C` | Копировать промпт |
| `Ctrl/Cmd + S` | Сохранить |

## 🔧 Настройка

### Добавление новых промптов

Промпты загружаются из структуры в `app.js`:

```javascript
const promptLibrary = {
    core: [
        {
            name: 'Название',
            file: 'filename.md',
            description: 'Описание'
        }
    ]
};
```

### Кастомизация темы

Измените CSS переменные в `styles.css`:

```css
:root {
    --primary-color: #6366F1;    /* Основной цвет */
    --bg-primary: #0F172A;       /* Фон */
    --text-primary: #F1F5F9;     /* Текст */
}
```

## 🚀 Развертывание

### GitHub Pages

```bash
# В настройках репозитория:
# Settings > Pages > Source: main branch / docs folder

# Скопируйте editor в docs
cp -r editor docs
git add docs
git commit -m "Deploy editor"
git push
```

### Netlify / Vercel

1. Подключите репозиторий
2. Build command: (не требуется)
3. Publish directory: `editor`
4. Deploy!

### Собственный сервер

```bash
# Nginx
cp -r editor /var/www/html/prompts

# Apache
cp -r editor /var/www/html/prompts
```

## 🐛 Известные проблемы

1. **Загрузка файлов** - в текущей версии используются демо-данные
   - Решение: требуется бэкенд или file system API

2. **Автосохранение** - пока только localStorage
   - Решение: интеграция с GitHub API

3. **Мобильная версия** - требует оптимизации
   - Решение: в разработке responsive layout
