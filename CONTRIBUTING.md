# Контрибьюция в Библиотеку Промптов для Вайбкодеров

Спасибо за интерес к проекту! Мы рады любому вкладу.

## Как внести вклад

### 1. Добавление нового промпта

Хороший промпт должен:
- ✅ Решать конкретную задачу
- ✅ Иметь понятное описание и примеры
- ✅ Включать use cases (когда использовать)
- ✅ Содержать примеры кода
- ✅ Указывать best practices

### 2. Структура промпта

Используйте этот шаблон:

```markdown
# Название Промпта - Краткое Описание

## Назначение
Что делает этот промпт и какую проблему решает

## Промпт

\`\`\`
[Сам промпт который можно скопировать]
\`\`\`

## Когда использовать

✅ **Используйте когда:**
- Случай 1
- Случай 2

❌ **Не используйте когда:**
- Случай 1
- Случай 2

## Примеры использования

### Пример 1: [Описание]

**Запрос:**
\`\`\`
[пример запроса]
\`\`\`

**Результат:**
\`\`\`
[пример результата]
\`\`\`

## Best Practices

[Рекомендации по использованию]

---

**Теги:** #tag1 #tag2 #tag3
```

### 3. Категории промптов

Поместите промпт в соответствующую категорию:

- `prompts/core/` - Основные промпты для ежедневной работы
- `prompts/testing/` - Промпты для тестирования
- `prompts/performance/` - Оптимизация и производительность
- `prompts/documentation/` - Документация и комментарии
- `prompts/ui/` - UI/UX и дизайн
- `prompts/specialized/` - Специализированные (БД, API, безопасность)

Если не уверены в категории - создайте промпт, мы поможем с размещением.

### 4. Процесс контрибьюции

1. **Fork** репозитория
2. **Создайте ветку** для вашей фичи:
   ```bash
   git checkout -b feature/my-awesome-prompt
   ```
3. **Добавьте промпт** в соответствующую категорию
4. **Протестируйте** промпт с реальным AI ассистентом
5. **Commit** с понятным сообщением:
   ```bash
   git commit -m "Add: [Название промпта]"
   ```
6. **Push** в ваш fork:
   ```bash
   git push origin feature/my-awesome-prompt
   ```
7. **Создайте Pull Request** с описанием:
   - Что делает промпт
   - Какую проблему решает
   - Примеры использования

## Требования к качеству

### ✅ Хороший промпт:
- Протестирован на реальных задачах
- Имеет 2-3 практичных примера
- Включает edge cases
- Понятен новичкам
- Содержит troubleshooting секцию

### ❌ Плохой промпт:
- Слишком общий ("напиши хороший код")
- Нет примеров использования
- Копирует существующий промпт
- Содержит маркетинговую шумиху
- Не протестирован

## Примеры хороших контрибьюций

### Новый промпт
```markdown
# Database Migration - Безопасные Миграции БД

## Назначение
Создание безопасных, обратимых миграций базы данных...

[полное описание с примерами]
```

### Улучшение существующего
```markdown
## Пример 3: Edge Case с большими данными

**Запрос:**
Как оптимизировать функцию для 1M+ записей?

**Решение:**
[практичное решение]
```

### Исправление ошибки
```markdown
# В промпте test-coverage.md
- Строка 45: исправлена опечатка в примере
- Добавлен пример для async тестов
```

## Code Style

### Markdown
- Используйте заголовки `##` для секций
- Code блоки с указанием языка: \`\`\`javascript
- Emoji умеренно (✅ ❌ ⚠️ 🔴 🟡 🟢)

### Примеры кода
- Должны быть runnable (работать без изменений)
- Включайте комментарии для сложных частей
- Показывайте и правильный ✅ и неправильный ❌ код

### Язык
- Основной язык: русский
- Технические термины можно на английском
- Примеры кода: английский (переменные, комментарии)

## Review Process

После создания PR:
1. Автоматические проверки (форматирование, ссылки)
2. Review от мантейнеров (1-3 дня)
3. Возможные правки
4. Merge в main
