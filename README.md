# Библиотека Промптов для Вайбкодеров 🚀

Коллекция профессиональных промптов для эффективного программирования с использованием AI-ассистентов (Claude, Cursor, GitHub Copilot, и др.)

## Что такое Вайб-кодинг?

**Вайб-кодинг** (Vibe Coding) — современный подход к разработке, при котором программист описывает задачу на естественном языке, а AI-ассистент генерирует код. Термин введён Андреем Карпатым (ex-Tesla AI Director) в 2025 году.

### Ключевые принципы:
- 🎯 Описывайте **что** нужно сделать, а не **как**
- 🔄 Итеративный подход: промпт → код → проверка → уточнение
- 🛡️ Контроль над изменениями и понимание сгенерированного кода
- 📚 Использование проверенных промптов для типовых задач

## Быстрый старт

### 1. Для начинающих
Начните с промпта **Overview First** - AI будет сначала объяснять, что собирается сделать:

```
Ты опытный разработчик. Перед генерацией кода всегда:
1. Объясни, что ты собираешься сделать
2. Спроси подтверждение
3. Только после одобрения генерируй код
```

### 2. Для контроля качества
Используйте **Guardrail Prompt** для точечных изменений:

```
Вноси ТОЛЬКО запрошенные изменения. Не изменяй другие части кода.
Перед изменением покажи diff того, что будет изменено.
```

### 3. Для создания функций
**Feature Builder** для сложных задач:

```
Создай функцию поэтапно:
1. Продумай архитектуру
2. Напиши интерфейсы/типы
3. Реализуй основную логику
4. Добавь обработку ошибок
5. Напиши тесты
```

## Лучшие практики

### ✅ DO:
- Начинайте с описания контекста и роли ("Ты senior React разработчик...")
- Разбивайте сложные задачи на этапы
- Проверяйте и понимайте сгенерированный код
- Используйте промпты для создания тестов
- Итерируйте: промпт → результат → уточнение

### ❌ DON'T:
- Не используйте код не понимая его
- Не пропускайте тестирование
- Не генерируйте большие фичи одним промптом
- Не забывайте про безопасность
- Не полагайтесь только на AI для критических решений

## Примеры использования

### Пример 1: Создание React компонента
```
Ты senior React разработчик, пишущий чистый, хорошо документированный код.
Создай компонент TodoList на TypeScript с:
- Полем ввода и кнопкой добавления
- Списком задач с чекбоксами
- Возможностью удаления
- Используй функциональные компоненты и хуки
- Добавь подробные комментарии к каждой функции
```

### Пример 2: Отладка производительности
```
Проанализируй этот код на производительность:
[ваш код]

Найди:
1. Узкие места (bottlenecks)
2. Неоптимальные алгоритмы
3. Лишние ререндеры (для React)
4. Утечки памяти

Предложи оптимизации с объяснением trade-offs.
```

### Пример 3: Генерация тестов
```
Создай comprehensive test suite для этой функции:
[ваш код]

Включи:
- Happy path тесты
- Edge cases
- Error handling
- Граничные значения
- Используй Jest/Vitest
```
